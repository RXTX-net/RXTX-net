## 10. Сообщество и mesh

**Где:** Сообщество → **Чат** (внутри вкладки: Чат / RXTX-NET / Конструктор / Рынок)

### 10.1 Чат

- **Global** — общий канал mesh
- **Private** — личные сообщения peer

не публикуйте seed и пароли в чат. Доверяйте peer явно (approve).

### 10.2 RXTX-NET

Обзор сети: peers, onion-адреса, статус mesh.

### 10.3 Конструктор (RXLang)

Страницы mesh — витрины, услуги.

**Пример страницы:**
```rxsrc
@title Анонимный хостинг
@section Тарифы
@text VPS в mesh, без clearnet DNS
@wallet btc:bc1qxxxxxxxx
@link rxtx://YOUR_FP/shop Заказать
@escrow lst-001
```

Вставьте в Конструктор → **Опубликовать** (нужен активный хостинг-слот).

Подробнее: [RXLANG.md](RXLANG.md)

### 10.4 Рынок

Листинги с **escrow** — сделки между узлами без прямого перевода незнакомцу.

**Пример:** продаёте слот хостинга → создаёте листинг → покупатель платит в escrow → вы подтверждаете → средства разблокированы.

## 11. Аккаунт, оплата, NetVault

**Где:** Система → **Вход RXTX**

### Seed и идентичность

Seed — ваша долгосрочная идентичность в RXTX-NET.  
**Храните офлайн.** Не в облаке, не в скриншотах.

### Оплата хостинга

BTC/ETH → баланс → тарифы слотов публикации страниц.

| Тариф (пример) | Слоты | Срок |
|----------------|-------|------|
| Starter | 1 страница | 30 дней |
| Pro | 3 + зеркало | 90 дней |
| Max | 10 + зеркала | 365 дней |

за 7 дней до окончания — продлите, иначе страницы снимут с публикации (данные останутся в vault).

### NetVault

Резервные копии узла, синхронизация между устройствами (на лицензированных нодах).

**Пример:** раз в неделю → NetVault → **Создать снимок** → сохранить на внешний носитель.

## 12. Дистанционное управление

**Где:** Безопасность → **Дистанционное**

| Канал | Для чего |
|-------|----------|
| Telegram-бот | Статус, reboot, логи |
| Discord-бот | То же |
| Webhook | n8n, Home Assistant, свои скрипты |
| REST API + JWT | Автоматизация |

**Пример webhook:** cron на VPS шлёт POST на `/api/remote/hook` с secret → узел отвечает статусом VPN.

webhook secret — длинная случайная строка. Не открывайте панель в clearnet без VPN.

## 13. Система и обслуживание

**Где:** Система → **Система**

| Действие | Когда |
|----------|-------|
| Смена пароля | Раз в 90 дней или после Panic |
| Умная очистка | Раз в неделю (кэш, temp — не трогает VPN/пароли) |
| Speedtest | После смены VPN/Tor |
| Reboot | После обновления конфигов сети |
| Kill System | Аварийное полное выключение сервисов |

**Вентилятор:** если железо с датчиком (Pi и др.) и температура выше 70°C — **Fan config**.

## 14. Готовые сценарии

### Сценарий A: «Безопасный браузинг»

1. VPN → подключить DE
2. DNS → через VPN
3. Leak Monitor → всё зелёное
4. Workstation → browser + AntiDetect профиль
5. Proxy → SOCKS в профиле браузера

### Сценарий B: «RXTX как роутер для телефона»

1. Internet → ethernet in, wifi out
2. VPN на wan
3. Wi‑Fi → AP включён
4. Телефон → Wi‑Fi RXTX → весь трафик через VPN

### Сценарий C: «Витрина в mesh»

1. Аккаунт → оплатить хостинг Starter
2. Конструктор → вставить `.rxsrc`
3. Опубликовать
4. Ссылка вида `rxtx://<fingerprint>/<id>` — в чате или на другой странице

### Сценарий D: «Уехал — включил Panic»

1. Монитор → **PANIC**
2. Физически обесточить при необходимости
3. На месте → Release → смена пароля → проверка логов

## 15. Частые вопросы

**Панель не открывается**  
Проверьте IP в роутере, кабель Ethernet, подождите 3 мин после boot.

**VPN подключён, но IP старый**  
DNS не через VPN → раздел DNS. Перезапуск сети в Системе.

**Lite mode — нет браузера в Docker**  
Только терминал. Полный browser — на образе с Docker.

**Страница не публикуется**  
Проверьте слот хостинга в Аккаунте и срок лицензии.

**Забыл пароль**  
Только повторная активация / перепрошивка (данные на узле будут потеряны).
