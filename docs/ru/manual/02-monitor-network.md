## 5. Монитор — главный экран

**Где:** Операции → **Монитор** (первая страница `/`).

### 5.1 Privacy Health

Оценка 0–100. Смотрите каждый день.

| Оценка | Что делать |
|--------|------------|
| 80–100 | Норма |
| 50–79 | Проверьте VPN, DNS, Tor |
| ниже 50 | Не работайте в открытую — настройте связку VPN+Proxy |

### 5.2 Leak Monitor

Проверяет четыре типа утечек:

| Проверка | Значение |
|----------|----------|
| DNS | Запросы идут через ваш туннель, а не провайдера |
| IPv6 | IPv6 не «обходит» VPN |
| VPN route | Весь трафик идёт через VPN |
| IP | Внешний IP совпадает с ожидаемым |

**Пример:** подключили VPN в Германии — Leak Monitor должен показать DE IP и зелёный DNS. Если DNS красный → зайдите в **DNS** и выберите пресет «через VPN».

### 5.3 Карта сети

Схема: Internet → VPN/Proxy → Tor → Docker → Mesh.  
Помогает понять, куда уходит трафик сейчас.

### 5.4 Panic Button

**Что делает:** мгновенная блокировка (lockdown).  
**Что НЕ делает:** не стирает диск.

**Когда нажимать:** подозрение на физический доступ, тревога, нужно обрубить сессии.

**Пример:** нажали PANIC → отпустили только после `Release block` и смены пароля.

### 5.5 Auto-Rotate Identity

Меняет связный пакет: страна, DNS, locale для Docker, эмиссия Wi‑Fi/BT.

| Настройка | Значение |
|-----------|--------------|
| Интервал | 30–120 мин для активной работы |
| Rotate now | Перед чувствительной сессией в браузере |

**Важно:** язык панели (RU/EN) **не меняется** — меняется только «внешнее лицо» для контейнеров.

### 5.6 Вкладка «Связки VPN/Proxy»

Готовые цепочки по странам: сканируете endpoint → привязываете к `wan` / `docker` / `browser`.

**Пример настройки:**
1. Монитор → вкладка **Связки VPN/Proxy**
2. Страна: `DE`
3. **Scan** → выбрать endpoint с низкой задержкой
4. Target: `wan` → **Apply**
5. Вернитесь на Монитор — Leak Monitor должен стать зелёным

### 5.7 Лицензия и Shield

- **Лицензия** — дни до окончания. За 7 дней — жёлтое предупреждение. Сохраните данные заранее.
- **Shield** — статус защиты узла. Если есть алерты — откройте **Система → Очистка** или перезагрузите сеть.

## 6. Сеть: VPN, Proxy, Firewall

### 6.1 VPN

**Где:** Сеть → **VPN**

Поддерживается: **OpenVPN**, **WireGuard**, **ProxyGuard**.

**Пример OpenVPN:**
1. Скачайте `.ovpn` от провайдера
2. VPN → тип OpenVPN → **Загрузить файл**
3. Выберите файл в списке → **Подключить**
4. Бейдж станет «Подключено»
5. Монитор → проверьте Leak Monitor

**Пример WireGuard:**
1. Получите `.conf` от провайдера
2. VPN → WireGuard → загрузить → подключить


- Один активный VPN на `wan` — не включайте два конфига сразу
- Именуйте файлы понятно: `de-frankfurt.ovpn`, `nl-amsterdam.conf`
- Перед сменой страны — **Отключить**, потом новый конфиг

### 6.2 Proxy

**Где:** Сеть → **Proxy**

SOCKS4/5, HTTP. Списки, проверка живости, связка с redsocks.

**Пример:**
```
Тип: SOCKS5
Хост: 127.0.0.1  Порт: 1080  (локальный после VPN)
```

**Сценарий VPN + Proxy:**
1. Подключите VPN (раздел 6.1)
2. Proxy → добавьте SOCKS поверх VPN
3. AntiDetect Browser → укажите этот proxy в профиле

### 6.3 Firewall

**Где:** Сеть → **Firewall**

**Быстрое правило — пример:**
- Действие: `DROP`
- Протокол: `tcp`
- Порт: `22`  
→ закрыть SSH с WAN (если не нужен снаружи)

**Профили:** сохраните набор правил как `home`, `paranoid`, `mesh-only` и переключайте.


1. Политика INPUT: `DROP` по умолчанию
2. Разрешить только LAN (192.168.x.x) на панель
3. Исходящий трафик — `ACCEPT` (пока настраиваете VPN)
