## 7. Wi‑Fi, DHCP, DNS, Internet

### 7.1 Internet (маршрутизация)

**Где:** Операции → **Internet**

Настраивает: вход (Ethernet/USB modem) → выход (Wi‑Fi AP / Ethernet) + NAT.

**Пример «роутер для телефона»:**
| | |
|---|---|
| Вход | `ethernet` (кабель провайдера) |
| Выход | `wifi` (точка доступа) |
| Действие | **Настроить маршрутизацию** |

После — телефон подключается к Wi‑Fi RXTX и выходит в интернет через ваш VPN.

### 7.2 Wi‑Fi

**Где:** Сеть → **Wi‑Fi**

- Поднять точку доступа (AP)
- Сканировать эфир
- Выбрать адаптер

SSID не персонализируйте — используйте нейтральное имя или Stealth-профиль.

### 7.3 DHCP

**Где:** Сеть → **DHCP**

Раздаёт адреса клиентам AP (подсеть 192.168.220.0/24).  
Обычно работает автоматически после включения AP.

### 7.4 DNS

**Где:** Сеть → **DNS**

Пресеты: системный, через VPN, Tor, кастом.

**Пример:** VPN подключён → DNS → пресет «через VPN» → **Применить** → Leak Monitor → DNS зелёный.

## 8. Tor и Stealth

### 8.1 TOR

**Где:** Безопасность → **TOR**

Включает выход в интернет через Tor. Мосты — для стран с блокировками.

Tor поверх VPN (сначала VPN, потом Tor) — для жёстких условий. Проверяйте скорость на **Система → Speedtest**.

### 8.2 Stealth

**Где:** Безопасность → **Stealth**

Geo-профили для «окружения» устройства:

**Пример профиля:**
| Поле | Значение |
|------|----------|
| Название | `cafe-us` |
| Страна | `US` |
| Fake SSID | `Starbucks WiFi` |
| MAC wlan0 | случайный |

создайте 3–5 профилей под разные сценарии (дом, кафе, аэропорт) и включайте через Auto-Rotate на Мониторе.

## 9. Workstation и AntiDetect Browser

### 9.1 Workstation

**Где:** Операции → **Workstation**

Docker-сессии: **терминал** или **браузер**.

| Режим | Когда |
|-------|-------|
| Терминал | SSH, curl, утилиты — без следов на хосте |
| Браузер | Веб с изоляцией |

**Пример сессии:**
1. Workstation → **Создать**
2. Тип: `terminal`, имя: `work-1`
3. **Открыть** → работаете в контейнере
4. Закрыли без «Save» → контейнер удалён

для любой работы в clearnet — только Docker-браузер, не хост.

### 9.2 AntiDetect Browser

**Где:** Операции → **AntiDetect**

Профили fingerprint (user.js). Каждый слот — свой отпечаток + proxy.

**Пример:**
1. Создайте профиль `shop-us`
2. Привяжите SOCKS из раздела Proxy
3. Запустите из Workstation → тип browser
